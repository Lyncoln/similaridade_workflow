  3. Propagation of type II associated CME using SHEBA and solar wind properties from ACE

    Created by Eoincar on Monday 17 February 2014 14:51:22 (UTC)

    Revision comment:

    This workflow firstly selects the fastest CME from the wind_typeii_soho_cme catalogue of the heliophysics event catalogue (HEC). It then finds properties of this CME from soho_lasco_cme and properties of the associated flare from goes_sxr_flare. The CME and flare properties are used to run the SHEBA propagation model to find an ETA range at Earth. The average solar wind speed at this time range is then used to re-define the CME speed (this accounts for solar wind drag). The new CME speed is then used in a second run of the SHEBA propagation model to find an ETA at Earth (or any chosen destination). This workflow is based on the design of 

    when_do_the_fastest_type_ii_cme_should_be_detected_throughout_all_the_heliosphere___returns_single

    This now has inputs of start time, end time and destination at which ETA is to be calculated
